# 1bit_LLM — MacBook Air 8GB で動かす軽量1ビットLLM

## 概要

MacBook Air (8GB RAM) 上で動作する1ビット量子化LLMの実験プロジェクト。
主に **Bonsai 8B** モデルを用いて、ローカル推論の可能性を探る。

## 対象モデル

| モデル | パラメータ | サイズ | 速度（Prompt） | 速度（Generation） | Tool Calling |
|--------|-----------|--------|--------------|------------------|-------------|
| **Bonsai 8B（自機実測）** | 8.2B | 1.1GB | **35.3 tok/s** | **27.1 tok/s** | ✅ |
| Bonsai 8B（記事参考値） | 8.2B | 1.1GB | — | 21.1 tok/s | ✅ |
| SwiftLM Qwen 2.5 3B | 3.1B | 1.7GB | — | 27.3 tok/s | ❌ |
| Ollama Qwen 2.5 3B | 3.1B | 2.3GB | — | 23.1 tok/s | ✅ |

- **Bonsai 8B** — ベースモデル: **Qwen3**。1ビット量子化（Q1_0_g128）で8.2Bパラメータを1.1GBに圧縮（圧縮率93%）。

## 量子化フォーマット

- 使用フォーマット: `Q1_0_g128`（1ビット量子化）
- 標準の Ollama (llama.cpp) は未対応 → **PrismML フォーク版 llama-server** が必要

## ローカルAIエージェントとしての能力評価

Continue + VS Code 環境（local_pygen）でのテスト結果（2026-04-06実測）。

| 操作 | 結果 | 備考 |
|------|------|------|
| ファイル新規作成 | ✅ 可能 | Continue `Ctrl+I` で指示するとそのまま作成できる |
| 既存ファイルの直接書き換え | ✅ 可能 | Ctrl+I インライン編集で安定動作 |
| 簡単なコード生成 | ✅ 良好 | 短い関数・スクリプト程度なら品質問題なし |
| Tool Calling | ✅ 動作する | 都市名など日本語で渡す点に注意 |
| 長いコード生成（100行超） | ❌ 苦手 | ハルシネーション多発・途中切断 |
| 一般知識・固有名詞の回答 | ❌ 苦手 | 1ビット量子化による知識欠落が顕著 |
| ネット参照が必要な質問 | ❌ 不可 | ローカルLLMはインターネット接続なし |

## 同時実行禁止

llama-server は処理中に別リクエストが来ると **JSON が崩れてエラー**になる。

```
❌ 禁止: Continue + local_agent CLI を同時に使う
❌ 禁止: 別ターミナルから同時にリクエストを送る
✅ 推奨: 1つのアプリ・ターミナルからのみ使う
```

エラー例: `Expecting value: line 1 column 1 (char 0)` → 競合による空レスポンスが原因。

## 推奨ユースケース

- **日本語文章生成**: Bonsai 8B（qwen2.5:7b を上回る品質）
- **コード補完・修正**: Bonsai 8B（短い関数・スクリプト程度）
- **エージェント用途**: Bonsai 8B（Tool Calling 対応、ファイル作成・書き換え可能）
- **速度重視**: SwiftLM Qwen 2.5 3B（ただし Tool Calling 非対応）
